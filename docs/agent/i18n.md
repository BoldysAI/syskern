# docs/agent/i18n.md — Multilingue & traduction (CDC §10)

> Domaine i18n : contenus produit multilingues (FR/EN/ES), service DeepL + cache,
> couverture catalogue, traduction bulk, résolution de la langue des offres.
> Règles transverses → `/AGENTS.md`. Clients HTTP externes → `integrations.md`.
> Async → `celery-task.md`. Réf code : `apps/i18n/`, `apps/offers/services/translation.py`,
> `apps/offers/services/offer_i18n.py`, `frontend/src/components/MultilingualField.tsx`.

Langue source = **FR** (fait foi, CDC §10.2). EN/ES dérivées (saisie manuelle ou DeepL).
Contenus multilingues = JSONB `{"fr","en","es"}` (`Product.description_marketing`/
`description_technical`, `AttributeRegistry.label`). UI de la plateforme = FR only.

## Service de traduction (CDC §10.4)

- **Client** : `apps/offers/services/translation.py::DeepLClient` (dans `offers`, pas
  `app/clients/` — cf. `decisions.md`). `translate()` + `translate_batch()`.
  `preserve_formatting=1` sur chaque appel ; `formality=more` **uniquement** si la langue
  cible le supporte (ES oui, EN non — sinon DeepL répond 400). Après DeepL, la casse du
  texte source est réalignée (`apply_source_casing`) pour conserver ALL CAPS / Title Case
  anglais lors d'un EN→FR (sinon `formality=more` normalise en typographie française).
  Erreurs typées :
  timeout 10 s), `TranslationInputError` (>5000 car.), `TranslationError` (auth/config).
  401/403 → log + email (`TRANSLATION_AUTH_ALERT_RECIPIENTS`).
- **Auth & hôte DeepL** (gotchas dev, cf. `decisions.md`) :
  - Auth = en-tête `Authorization: DeepL-Auth-Key <clé>` — **plus** de `auth_key` dans le body
    (legacy → HTTP 403 trompeur « authentication failure »).
  - Hôte auto : clé Free (suffixe `:fx`) → `https://api-free.deepl.com/v2` ; clé Pro →
    `https://api.deepl.com/v2`. Surcharge optionnelle `DEEPL_API_URL` dans `.env`.
  - Clé absente → `503` « Service de traduction non configuré… » (pas de 500).
- **Variables d'env** (`backend/.env`) : `DEEPL_API_KEY` (requis pour traduire),
  `DEEPL_API_URL` (optionnel), `TRANSLATION_CACHE_TTL_DAYS` (défaut 90),
  `TRANSLATION_AUTH_ALERT_RECIPIENTS`. Après changement de clé → redémarrer le backend ;
  traduction bulk → redémarrer Celery aussi.
- **Cache** : app **`apps/i18n`** (label `app_i18n`), modèle `TranslationCache`
  (`db_table="translation_cache"`) : `source_text_hash` UNIQUE (sha256 `src|tgt|text`),
  `expires_at`, `hit_count`. TTL `TRANSLATION_CACHE_TTL_DAYS` (défaut 90).
  Service read-through : `apps.i18n.services.translate_cached(text, src, tgt) -> (texte, from_cache)`
  et `translate_many_cached(...)` (batch, regroupe les miss). **Écart CDC §10.4.3** (table
  dédiée vs JSONB) tracé dans `decisions.md` ; le JSONB produit reste la source des fiches.
- **Endpoint** : `POST /api/translate` (`apps/i18n/views.TranslateView`) — **synchrone**
  (écart règle §4, cf. `decisions.md`). Body `{text | texts[], source_lang, target_lang}`
  (batch cap 50). Réponse `{translated_text, from_cache}` ou `{translations:[…]}`.
  `456`→402, `503`→503 (quota/indispo/auth/config), trop long→400. Le front appelle **toujours**
  cet endpoint, jamais DeepL directement. `MultilingualField` affiche le `detail` API en cas d'erreur.
- **Purge** : Beat `i18n.purge_translation_cache` (03:30 UTC, migration `app_i18n/0002`).

## Frontend multilingue

- **`components/MultilingualField.tsx`** (réutilisable) : 3 onglets FR/EN/ES, input **ou**
  textarea (`kind`), point d'état par onglet (rempli=vert / vide=ambre + `sr-only`),
  bouton « Traduire depuis FR » (EN/ES) → `translateText()` (`/api/translate`, remplit
  l'onglet, éditable). `mode="read"|"edit"`, `requiredSource` marque FR d'un `*`.
  Props : `{ value:{fr,en,es}, onChange, mode, kind, label, requiredSource, rows }`.
  Utilisé par : `catalog/[sku]/_tabs/DescriptionsEditor` (descriptions), `settings/attributes/
  _components/AttributeFormModal` (label). **Notes fournisseur = hors scope** (texte simple).
- API (`lib/api.ts`) : `translateText`, `startBulkTranslate`, `getTaskStatus`,
  `checkOfferCoverage`, type `I18nCoverage`.

## Couverture multilingue (CDC §10.7.3)

- Backend : `Product.i18n_coverage` (`{languages, percent, complete}`) exposé sur
  `ProductListSerializer` + détail. Filtres catalogue :
  - `?i18n_incomplete=true` — au moins une des 3 langues manquante ;
  - `?lang_fr_in=true` / `lang_fr_out=true` (idem `en`, `es`) — avec / sans contenu dans
    descriptions marketing ou technique (`ProductFilter._lang_covered_q`).
- Front : colonne « Langues » toujours visible sur la page catalogue
  (`useCatalogColumns({ showLanguageColumn })`, largeur compacte ~76px) ;
  filtres sidebar : « Au moins une langue manquante » + par langue « Avec contenu » /
  « Sans contenu » (mutuellement exclusifs par langue). **Aucun libellé « i18n »** visible
  utilisateur. Chips/compteur via `active-filters.ts`.

## Traduction bulk (CDC §10.3.2)

- Backend : `products.bulk_translate_products_task` (bind, `update_state("PROGRESS",
  {current,total})`) — ne remplit que les langues **vides** (n'écrase jamais), abort clair
  sur quota/indispo. Endpoint `POST /api/products/bulk-translate/` (202 + task_id) ;
  serializer `BulkTranslateSerializer` (`ids, source_lang, target_langs, content_fields`).
- Front : `catalog/_components/BulkTranslateDialog` ouvert depuis la `selectionBar` du
  catalogue (`canEdit`) — choix langues/champs, progress bar (poll `/api/tasks/{id}/`), récap.

## Résolution langue offre (CDC §10.5)

- **Tarif par client** : `GenerateTariffOffersSerializer.language_per_client` (défaut false).
  `true` → chaque offre = `client.preferred_language or language or "fr"` (résolu dans
  `generate_tariff_offers_task`). Wizard tarif : toggle « Langue par client (automatique) ».
- **Projet** : wizard pré-remplit la langue avec `client.preferred_language` (surchargeable).
- **Fallback** (`services/offer_i18n.resolve_product_description`) : langue cible sinon FR
  (`fallback_used`) ; les replis sont **loggés** à la génération (`_log_language_fallbacks`).
- **Pré-génération** : `POST /api/simulations/{id}/offer-coverage-check/` (body `{language}`
  ou `{client_ids, language_per_client}`) → `{languages, products[], product_ids[]}`.
  UI : `components/OfferCoverageWarning` (dans les 2 wizards) — warning + « Traduire
  automatiquement » (bulk translate des `product_ids` puis re-check).
